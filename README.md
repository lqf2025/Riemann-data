# Figure Generation Repository

This repository is used to reproduce four figures:

* **Fig. 3**
* **Fig. 4**
* **Ex_Fig. 1**
* **Ex_Fig. 2**

## Repository structure

```text
.
├── data/
│   ├── data.py
│   ├── data2.py
│   ├── data.npz
│   ├── data2.npz
│   └── *.mat
├── Fig3.py
├── Fig4.py
├── Ex_fig1.py
├── Ex_fig2.py
├── GLA.py
├── GLA1.npz
├── GLA2.npz
├── GLA3.npz
└── README.md
```

## Overview

The main purpose of this repository is to reproduce the four figures listed above.

* **Fig3.py** and **Fig4.py** mainly depend on the `data/` folder.
* Inside `data/`, **`data.npz`** and **`data2.npz`** are generated numerical data files.
* The other files in `data/` are experimental or preprocessed input data.
* **Ex_fig2.py** does not rely on additional generated data files.
* **Ex_fig1.py** depends on **`GLA1.npz`**, **`GLA2.npz`**, and **`GLA3.npz`**, which are generated by **`GLA.py`**.

## Requirements

A standard scientific Python environment is sufficient. The scripts mainly use:

* `numpy`
* `matplotlib`
* `scipy`
* `mpmath`
* `multiprocessing`
* `warnings`
* `pathlib`

## How to generate each figure

### 1. Fig. 3

Run:

```bash
python Fig3.py
```

This generates:

```text
Fig3.pdf
```

### 2. Fig. 4

Run:

```bash
python Fig4.py
```

This generates:

```text
Fig4.pdf
```

### 3. Ex. Fig. 2

Run:

```bash
python Ex_fig2.py
```

This generates:

```text
Ex_fig2.pdf
```

This script does not require additional generated data files.

### 4. Ex. Fig. 1

This figure requires three generated files:

* `GLA1.npz`
* `GLA2.npz`
* `GLA3.npz`

These files are produced by:

```bash
python GLA.py
```

After they are generated, run:

```bash
python Ex_fig1.py
```

## About the `data/` folder

The `data/` folder contains two kinds of files.

### Generated numerical data

These are intermediate numerical results used by the plotting scripts:

* `data.npz`
* `data2.npz`

If needed, they can be regenerated by running:

```bash
cd data
python data.py
python data2.py
cd ..
```

### Experimental / input data

All remaining `.mat` files in `data/` are experimental data or processed input data used by the plotting scripts.

## Reproducibility workflow

To reproduce all figures from scratch:

1. Install the required Python packages.
2. Make sure the `data/` folder is present.
3. If `data.npz` or `data2.npz` are missing, regenerate them using `data/data.py` and `data/data2.py`.
4. Run `Fig3.py`, `Fig4.py`, and `Ex_fig2.py` directly.
5. For `Ex_fig1.py`, first run `GLA.py` to generate `GLA1.npz`, `GLA2.npz`, and `GLA3.npz`, then run `Ex_fig1.py`.
